jot (Joy Of Text) - a text editor

Jot is a collection of a few basic text editing functions wrapped up in a terse programming language with a customizable user interface and highly capable viewer. The default startup script sets up a useful selection of functions and reasonably-sane function-key environment.

Anyway, you might well ask, why should we be needing yet another text editor when there are already dozens to chose from? Well one good reason is that this one is better than the others, being simpler, speedier, smaller, more powerful, more flexible, less crashey and much more fun than many of it's main rivals.

In comparison to the three main open-source rivals: emacs, vim and nano. Not only does jot match the power, sophistication and elegance of emacs without requiring postgraduate degrees in pure and applied emacs but it also matches the simplicity and can-do usefulness of vim without the usual RSI risks. But what, I hear someone ask, about nano? Well, if anyone's still bothered about the sizes of executable in this day and age, the jot executable is significantly smaller than nano's. More importantly, jot is nothing like as crashy as nano.

The default startup script defines a powerful although generalist set of functions. When it is necessary to go off piste, command strings can be typed in without entering any special mode. The jot language consists of about 40 terse commands and a similar number of more specialised ones for housekeeping operations. Then it's just programming - a skill many text-editor users claim to possess anyway.

Jot came about in response to a requirement to analyze large intermediate CAD files, reports and source files. For these it's frequently necessary to cross-reference two or more files (maybe a violation report, a netlist and a parasitics file) to extract some nugget of truth, locate an inconsistency or identify elements mentioned in some report. That kind of activity demands a programming interface and very speedy, reliable text processing.

Although it's natural environment is chewing it's way through large CAD files, that power, speed and adaptability make it extremely useful for many other text-based applications. It's quite useful for documentation maintenance, it can generate simple htmls can interface with a spelling checker to generate reasonably tidy plaintext documents. It interfaces well with virtually any kind of command-driven agents - document-format converters, spelling checkers, compilers, debuggers, CAD systems ...

Main features:

  • A function-key and command driven text editor with a terse powerful command language - see jot programming.

  • Good support for tabular text especially spreadsheets - see about tabular text.

  • Good support for unicode - see about unicode.

  • Good support for large files - see about Large Files.

  • Good support for files with abnormally long lines - see about long lines.

  • Supports Exuberant-CTags-generated index files - see about large collections of files.

  • Strong interaction with system environment - can interface with virtually any command-driven agent.

  • A useful library of command scripts.

  • Easy construction of macro-commands and function-key mapping.

  • Minimal use of magic modes and similar trickery.

  • Hash-table support - allows indexing and speedy lookup in large files - see about hashtables.

  • Can perform operations on values picked up from text and push values back to text using C printf formatting.

  • Recovery of a session following a power failure or crash - see about journal files.

  • Has a useful document-processing facility - formats paragraphs, maintains headings, interfaces to an external spell-check agent, where spelling corrections are offered, these are displayed in menu form - see doc.jot.

  • Useful code-debugging features - see the jot debugger.

  • Supports various optional buffer locks - see about not losing your work

  • Screen can be allocated between any number of viewing areas - about multi-window working

  • Customized startup - start-up with any number of files pre-loaded into predetermined buffers and redefinition of function-key assignments - see about project files.

  • Supports metadata tagging - selected text can be highlighted in colour or can carry invisible packets of data - see about tagged text.
